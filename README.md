# 淘宝电商用户行为分析与增长优化项目

本项目基于淘宝公开用户行为数据，模拟电商平台数据分析师视角，围绕用户的浏览、收藏、加购、购买行为，构建“指标体系搭建 → 转化漏斗分析 → 活跃与留存分析 → 用户分群 → 类目机会识别 → SQL 验证 → Dashboard 展示”的完整数据分析流程。

项目目标不是单纯完成图表展示，而是从产品数据分析、增长分析和商业分析视角，识别平台转化瓶颈、用户运营机会和商品类目增长空间，并输出可落地的增长优化策略。

---

## 1. 项目背景

电商平台的增长不仅依赖新增流量，也依赖对存量用户行为链路的持续优化。用户在平台内通常会经历浏览、收藏、加购、购买等行为，不同行为对应不同的消费意图，也代表不同的运营机会。

本项目以淘宝用户行为日志为基础，模拟真实互联网业务分析场景，重点分析以下问题：

- 平台整体流量规模和用户规模如何？
- 用户从浏览到购买的路径中，哪些环节存在明显流失？
- 用户在一天中、一周中是否存在活跃规律？
- 新用户留存表现如何？
- 哪些用户具有更高运营价值？
- 哪些商品类目存在高浏览低购买、高加购低购买、高收藏低购买等增长机会？
- 如何针对不同用户群体和类目机会制定差异化增长策略？

## 2. 数据说明

数据集：Taobao User Behavior Dataset  
数据文件：`UserBehavior.csv`

原始数据包含 5 个字段：

| 字段 | 含义 |
|---|---|
| `user_id` | 用户 ID |
| `item_id` | 商品 ID |
| `category_id` | 商品类目 ID |
| `behavior_type` | 用户行为类型 |
| `timestamp` | 行为发生时间戳 |

行为类型说明：

| 行为类型 | 含义 |
|---|---|
| `pv` | 浏览商品详情页 |
| `fav` | 收藏商品 |
| `cart` | 加入购物车 |
| `buy` | 购买商品 |

说明：

- 原始 CSV 文件约 3.67GB，GitHub 仓库不上传原始数据。
- 复现项目时，请自行下载 `UserBehavior.csv`，并放置到 `data/raw/` 目录下。
- 数据中的 `category_id` 仅为数字编号，不包含真实类目名称，因此本项目不编造“美妆、服饰、食品”等类目名称，所有类目分析均以类目编号呈现。
- 数据不包含价格、订单金额、用户画像、活动曝光等字段，因此本项目不计算 GMV、客单价、收入贡献等收入类指标。

## 5. 分析流程

### 5.1 数据理解与抽样检查

由于原始数据文件较大，项目首先读取前 100 万行样本进行数据理解，检查字段结构、缺失值、重复值、行为类型分布和时间范围。

这一阶段主要用于确认数据能被正确读取，并完成时间戳转换、日期字段拆解等基础处理。

### 5.2 全量核心指标分析

在全量分析阶段，项目使用 `chunksize` 分块读取 3.67GB 原始 CSV，避免一次性读入内存导致程序崩溃。

核心指标包括：

- 总行为数
- 独立用户数
- 独立商品数
- 独立类目数
- PV 浏览量
- 收藏数
- 加购数
- 购买数
- 购买用户数
- 用户级购买覆盖率

其中，用户级购买覆盖率定义为：

```text
购买用户数 / 独立用户数
```

该指标表示在统计周期内，有购买行为的用户占全部活跃用户的比例，不等同于“单次浏览到购买的转化率”。

### 5.3 用户行为漏斗分析

漏斗分析从行为层面和用户层面分别展开：

- 行为层面：统计 `pv`、`fav`、`cart`、`buy` 的发生次数及转化关系。
- 用户层面：统计发生过浏览、收藏、加购、购买行为的去重用户数。

该模块用于识别用户从兴趣产生到最终购买之间的主要流失环节，为后续转化优化提供依据。

### 5.4 用户活跃与留存分析

本模块分析用户在不同时间维度下的活跃规律，包括：

- 每日活跃趋势
- 每小时活跃趋势
- 每周活跃趋势
- 新用户次日、3 日、7 日留存

该模块可以帮助平台判断用户活跃高峰，优化推荐触达、优惠券发放和活动推送时间。

### 5.5 用户分群与运营策略

项目基于用户行为特征构建用户级画像，包括：

- 浏览次数
- 收藏次数
- 加购次数
- 购买次数
- 活跃天数
- 互动商品数
- 互动类目数

在此基础上，采用规则法将用户划分为：

- 高价值用户
- 普通购买用户
- 高潜力用户
- 犹豫收藏用户
- 高浏览低转化用户
- 低活跃用户
- 其他用户

相比直接使用聚类算法，规则分层具有更强的业务解释性，更适合产品数据分析和用户运营场景。

### 5.6 商品类目机会分析

项目从类目维度统计浏览、收藏、加购、购买表现，并识别四类类目机会：

| 机会类型 | 业务含义 |
|---|---|
| 高浏览低购买类目 | 用户有兴趣，但购买转化不足 |
| 高加购低购买类目 | 用户接近购买，但临门转化不足 |
| 高收藏低购买类目 | 用户有兴趣但决策周期较长 |
| 高转化优势类目 | 类目购买意愿较强，适合重点推荐 |

本模块进一步结合用户分群结果，分析不同用户群体在重点类目中的行为贡献，从而形成“类目机会 × 用户群体”的增长策略。

### 5.7 SQL 业务查询与指标验证

项目将 `data/processed/` 下生成的结果表导入 SQLite，并使用 SQL 进行业务查询和指标验证。

SQL 模块主要覆盖：

- 核心指标查询
- 漏斗结果查询
- 用户分群查询
- 活跃趋势查询
- 类目机会查询
- 增长策略查询

在本项目中，Python 主要用于处理大文件、构建特征和生成分析结果；SQL 主要用于结构化查询、指标复现和业务口径验证。

### 5.8 Streamlit Dashboard 展示

项目使用 Streamlit 搭建可视化 Dashboard，展示以下模块：

- 项目概览
- 核心指标
- 转化漏斗
- 活跃与留存
- 用户分群
- 类目机会
- 增长策略

Dashboard 只读取 `data/processed/` 下已经生成的结果文件，不读取原始 3.67GB 数据，适合作为 GitHub 和简历项目展示入口。

## 6. 核心分析结果

### 6.1 平台整体规模

全量数据处理结果显示：

| 指标 | 数值 |
|---|---:|
| 总行为数 | 100,150,807 |
| 独立用户数 | 987,994 |
| 独立商品数 | 4,162,024 |
| 独立类目数 | 9,439 |
| PV 浏览量 | 89,716,264 |
| 收藏数 | 2,888,258 |
| 加购数 | 5,530,446 |
| 购买数 | 2,015,839 |
| 购买用户数 | 672,404 |
| 用户级购买覆盖率 | 68.06% |

### 6.2 行为结构

浏览行为占绝大多数，PV 约 8,971.6 万次，占总行为约 89.6%；购买行为约 201.6 万次，占总行为约 2.0%。

这说明平台拥有较大的流量基础，但从浏览到购买仍存在明显转化优化空间。

### 6.3 加购行为更接近购买决策

全量结果中，加购次数明显高于收藏次数。相比收藏行为，加购通常代表更强的购买意图，因此加购未购买用户和高加购低购买类目是后续运营转化的重点对象。

### 6.4 用户活跃具有时间规律

从时间维度看，用户活跃存在明显波动。活跃高峰可以用于指导推荐触达、优惠券发放和活动推送时间安排。

如果购买高峰与浏览高峰不完全一致，平台可以进一步区分“种草触达时间”和“转化刺激时间”，分别优化推荐和促销策略。

### 6.5 不同用户群体具有不同运营价值

用户分群结果显示，不同用户在购买次数、加购行为、收藏行为和活跃天数上存在明显差异。平台不应对所有用户采用同一种运营方式，而应基于用户行为阶段进行差异化触达。

### 6.6 商品类目存在结构性增长机会

类目机会分析识别出以下机会池：

- 高浏览低购买类目
- 高加购低购买类目
- 高收藏低购买类目
- 高转化优势类目

其中，高加购低购买类目适合通过优惠券、降价提醒、库存提醒等方式推动临门转化；高浏览低购买类目则更适合优化详情页、评价内容、价格展示和推荐匹配度。

## 7. 增长策略建议

### 7.1 用户运营策略

| 用户群体 | 用户特征 | 运营策略 |
|---|---|---|
| 高价值用户 | 多次购买且活跃度较高 | 会员权益、专属优惠、复购推荐、搭配推荐 |
| 普通购买用户 | 已购买但复购深度有限 | 相关商品推荐、满减券、组合推荐 |
| 高潜力用户 | 加购较多但尚未购买 | 限时优惠券、降价提醒、库存提醒 |
| 犹豫收藏用户 | 收藏较多但购买不足 | 买家秀、评价内容、相似商品推荐、收藏降价提醒 |
| 高浏览低转化用户 | 浏览多但没有购买 | 优化推荐精准度、详情页信息和价格展示 |
| 低活跃用户 | 行为次数少或活跃天数低 | 召回券、活动提醒、个性化推荐 |

### 7.2 类目运营策略

| 类目机会 | 业务问题 | 优化策略 |
|---|---|---|
| 高浏览低购买类目 | 流量高但购买弱，用户兴趣未转化为订单 | 优化详情页、价格展示、评价内容和推荐匹配度 |
| 高加购低购买类目 | 用户接近购买但最终流失 | 推送优惠券、满减、库存提醒和降价提醒 |
| 高收藏低购买类目 | 用户有兴趣但决策周期较长 | 推送买家秀、评价内容、相似商品对比 |
| 高转化优势类目 | 类目转化表现较好 | 面向高价值用户做复购推荐、搭配推荐和会员活动 |

- 使用 1 亿级用户行为日志进行分析，覆盖浏览、收藏、加购、购买完整行为链路。

## 10. 项目不足与后续优化

- 数据中 `category_id` 仅为数字编号，缺少真实类目名称，限制了类目解释深度。
- 数据缺少价格、订单金额、用户画像、活动曝光等信息，无法进一步分析 GMV、客单价、收入贡献和人群画像差异。
- 当前用户分群主要采用规则法，解释性强，但灵活性有限；后续可以尝试 KMeans 聚类作为补充分析。
- 当前策略主要基于历史行为数据推导，后续可以加入 A/B 测试，验证优惠券、降价提醒、推荐优化等策略的真实效果。
- 后续可以进一步构建购买预测模型或推荐系统模块，提升项目的建模深度。
